# Rejection Ledger — patterns considered and deliberately NOT adopted

One entry per rejected/adapted pattern, with the reason. Add here whenever a design is
considered and declined — future sessions read this instead of re-litigating.

| Pattern | Source | Ruling | Why |
|---|---|---|---|
| "Per item" as a standing-ruling scope unit | standing ruling #1, first night of autonomous execution (2026-07-14/15) | **Adapted** (2026-07-15) | Third spec-ambiguity instance in one mission ("adversaries in parallel" ≠ isolated; "verdict" naming two decisions; now "per item" readable as per-plan-item or per-adversary-verdict). The loop executed twice under the permissive reading and refused a third — correct under either reading, but the ambiguity forced a park and a human round-trip that invariant-grade wording would have avoided. Ruling rewritten to reading (a): one round per adversary-verdict per plan item, capped by bench size. **STANDING RULE: standing rulings get invariant-grade wording; the moment an ambiguity is found, the text is rewritten immediately — never re-litigated session by session.** |
| Waiving a substrate-class defect in the North Star instrument | run #4 arbitration option (c), 2026-07-14 | **Rejected** (2026-07-14) | "Substrate classes get reimplemented, not waived." Five DATA_INTEGRITY rounds proved instance-patching loses combinatorially against bash+jq stream semantics (multi-doc validation bypass, NUL stripping, exit-status quirks); a known-corruptible measurement instrument violates truth-only — the loop's credibility IS the operator's ability to trust the numbers. Ruling: reimplement on a substrate whose parsing succeeds-or-throws (zero-dependency Node, ratchet precedent), acceptance-gated on the full accumulated fixture corpus BEFORE the adversary re-runs, old substrate deleted in the same commit. |
| Installing pasted governance docs "verbatim", transmission damage included | corrupted clipboard pastes during the `/mission-loop` + `/objective-audit` installs (2026-07-14) | **Rejected** (2026-07-14) | Mid-word truncations in a paste are transport noise, not authorial intent; installing them into executable prompts degrades every future run. **Standard response ratified — union-reconstruction:** rebuild from the overlap of multiple pastes of the same document where available; repair only evident truncations, never redesign; disclose EVERY repair in a manifest for operator veto, with inferential repairs explicitly flagged as inferred. |
| One agent role-playing the RED_TEAM bench | gadd v0.1 cc adapter (`agents/gadd-redteam.md`: "5 adversaries reviewing ONE diff in parallel perspectives"; same collapse in lv `redteam.sh`'s single prompt) | **Rejected** (2026-07-14) | Spec §3 said "adversaries in parallel" but never said *isolated* — so the adapter collapsed the bench into one Opus context role-playing five perspectives. One context has one set of blind spots, and role-played adversaries produce correlated verdicts; the bench's value is uncorrelated failure detection, not persona prose. **Spec-ambiguity lesson:** when an invariant depends on isolation/independence, the spec must say so explicitly ("own isolated invocation, never sees another's verdict") or an adapter will optimize it away as mere phrasing. Fixed by splitting the bench into `RED_TEAM/` (one definition file per adversary + gate-matrix.md), dispatching each adversary as its own isolated parallel invocation in both adapters (cc: five `gadd-rt-*` subagents; lv: five independent API calls), mapping models structural→cheap / judgment→strong per the Phase 0 orchestration, and naming independence a spec concern in §3. |
| Adversary re-run session continuation (speed-audit P6/H5) | GADD SPEED AUDIT v1 proposal queue (local-private audit), operator ratification 2026-07-16 | **Rejected** (2026-07-16, per the auditor's own recommendation) | Continuing one adversary session across its own bench rounds would save an estimated 3–5% of tokens (~40% of adversary cache-writes) but trades adversary independence across rounds: an adversary that sees its prior verdict anchors on it, and fresh eyes each round is load-bearing for the arms race — the record of round-over-round NEW blocker classes (DATA_INTEGRITY reaching round 7, TEST_HONESTY 6 rounds including a flipped North-Star formula that survived 76/76 green assertions) is the receipt. Speed never buys down grader independence; the bench contract is grader territory, and this was one of only two queue items whose gain attacked a safety surface instead of pure overhead. |
| Instructions referencing payloads that live outside the repo | first draft of the seed self-application wave (2026-07-15) | **Rejected** (2026-07-15) | 4th spec-ambiguity instance, new subtype: "instructions must carry their payload verbatim or point to a readable source — never assume the executor knows text that lives outside the repo." Origin: the first draft of this very wave referenced a UBC document that existed only in operator-side conversation artifacts. Corollary proven in transit: the payload carrying this entry itself arrived damaged once, and the receiving auditor correctly refused to install its own unconfirmed reconstruction — the rule enforcing itself on its own delivery. |
